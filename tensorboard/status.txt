Summary
We have been upgrading the local TensorBoard 2.21 build to align with TensorFlow 2.21, mainly in two areas: Bazel 6.5.0 -> 7.7.0 and protobuf -> 6.31.1. The local source build is working, the pip wheel build is working, and the pip package smoke test is working. The main remaining gap is the full Bazel test suite.

What We Completed

Upgraded TensorBoard’s local Bazel version to 7.7.0.
Updated the workspace/version checks and Bazel-7 compatibility wiring in WORKSPACE, .bazelversion, .bazelrc, and related compatibility files.
Upgraded protobuf to 6.31.1 and aligned related deps like gRPC.
Reworked TensorBoard proto generation to avoid removed protobuf internals and stay compatible with newer Bazel/protobuf.
Fixed several repo-loading and analysis blockers from Bazel 7 / protobuf 6 migration.
Normalized CRLF-sensitive files so Linux Bazel builds no longer fail on shell/Starlark line endings.
Verified local source wheel build and install succeed.
Fixed bazel run //tensorboard/pip_package:test_pip_package -- --tf-version "tensorflow==2.21" so the installed wheel runtime works.
Fixed Docker-side Java setup again after moving closer to the CI base image.
Fixed the update_protos flow: bazel run //tensorboard/data/server:update_protos now refreshes descriptor.bin, and update_protos_test passes afterward.
Fixed the original Chrome shared-library blocker by improving the Docker image; libXrandr.so.2 errors are now gone.
